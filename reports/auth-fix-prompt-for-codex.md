**Задача:** привести регистрацию, логин и восстановление пароля на https://1001sovet.ru в production-ready состояние.

**Строгие правила:**
- Используй только beads (`bd create`, `bd update`, `bd close`). Создай эпик + подзадачи по приоритетам P0 → P1 → P2.
- Никогда не ломай продакшн, не удаляй реальных пользователей, не меняй пароли владельца.
- Тестовые аккаунты — только с префиксом `qa-test-delete-after-...` и обязательно фиксируй в `reports/test-accounts-and-seed-data.md` что и когда удалить.
- Сначала закрой все P0 (полноценный forgot/reset password + стабильная доставка писем + диагностика).
- Добавь show/hide password, confirm password, чекбокс согласия с условиями, валидацию силы пароля.
- Все ошибки — на русском, дружелюбные, без сырых сообщений Supabase и без утечки существования аккаунтов.
- Сделай переиспользуемый компонент PasswordInput.
- Почини/гарантируй создание записи в profiles при регистрации (лучше через trigger + миграцию).
- Добавь Turnstile на формы регистрации и входа.
- Проверь и при необходимости почини SMTP/DKIM/DMARC (координируй с владельцем).
- Добавь хотя бы базовые тесты и улучши видимость ошибок доставки писем.
- В конце обнови HANDOFF*.md и закрой бид.

**Порядок работы:**
1. Прочитай отчёт.
2. Создай/обнови бид(ы) с планом.
3. Реализуй по приоритетам.
4. После каждого значимого этапа — коммит + push + обновление beads.

Начинай немедленно с P0. Пиши коротко и по делу.
